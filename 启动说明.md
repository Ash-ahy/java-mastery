# 启动说明

## 项目位置

```powershell
C:\Users\aihey\Desktop\java_study\java-mastery
```

## 启动前提

本项目默认使用 `docker` profile，依赖：

- MySQL: `localhost:13306`
- Redis: `localhost:6380`
- JDK 17
- Maven

## 第一步：启动依赖

如果 Docker 容器还没启动，先执行：

```powershell
docker start mastery-mysql
docker start mastery-redis
```

如果容器不存在，再执行：

```powershell
docker compose up -d
```

这两个容器现在归属于同一个 compose 项目：

```text
java-mastery
```

也可以直接双击：

```text
docker_start.bat
```

## 第二步：进入项目目录

```powershell
cd C:\Users\aihey\Desktop\java_study\java-mastery
```

## 第三步：打包

```powershell
mvn package -DskipTests
```

如果你确认没有旧进程占用，也可以用：

```powershell
mvn clean package -DskipTests
```

如果 `clean` 失败，通常是旧的 `java-mastery-2.0.0.jar` 还在运行，占住了 `target` 目录里的 jar。

## 第四步：启动项目

### 方式一：前台启动，适合自己看日志

```powershell
java -jar target\java-mastery-2.0.0.jar --spring.profiles.active=docker
```

### 方式二：使用调试脚本

```text
run_debug.bat
```

### 方式三：重新构建并后台启动

```text
rebuild.bat
```

## 常见问题

### 1. `mvn clean package` 失败

原因：
旧的 Java 进程还在运行，占用了 `target\java-mastery-2.0.0.jar`

处理：

```powershell
Get-CimInstance Win32_Process | Where-Object { $_.CommandLine -like '*java-mastery-2.0.0.jar*' } | ForEach-Object { Stop-Process -Id $_.ProcessId -Force }
```

然后重新执行：

```powershell
mvn package -DskipTests
```

### 2. 8080 端口被占用

检查：

```powershell
netstat -ano | findstr :8080
```

如果确实被旧进程占用，先结束旧进程，再启动项目。

### 3. 登录一直卡住

先强制刷新浏览器：

```text
Ctrl + F5
```

然后重新登录。

### 4. Docker profile 启动失败

先确认依赖端口：

- MySQL: `13306`
- Redis: `6380`

再确认容器状态：

```powershell
docker ps
```
